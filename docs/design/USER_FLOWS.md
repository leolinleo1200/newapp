## 1. 學員註冊與登入

### 1.1 首次註冊流程

```mermaid
flowchart TD
    A[啟動 App] --> B{已登入?}
    B -->|是| C[進入 Dashboard]
    B -->|否| D[顯示 Welcome 畫面]
    D --> E[點擊「註冊」]
    E --> F[填寫註冊表單]
    F --> G[輸入 Email]
    G --> H[輸入密碼]
    H --> I[確認密碼]
    I --> J[輸入姓名]
    J --> K[選擇角色: 學員/教練]
    K --> L[點擊「註冊」]
    L --> M{驗證資料}
    M -->|失敗| N[顯示錯誤訊息]
    N --> F
    M -->|成功| O[呼叫 POST /auth/register]
    O --> P{API 回應}
    P -->|失敗| Q[顯示錯誤: Email已存在]
    Q --> F
    P -->|成功| R[儲存 JWT Token]
    R --> S[顯示「註冊成功」]
    S --> T[進入個人資料設定]
    T --> U[輸入生日]
    U --> V[選擇性別]
    V --> W[選擇主要泳姿]
    W --> X[點擊「完成」]
    X --> Y[呼叫 PUT /users/profile]
    Y --> C
```

### 1.2 登入流程

```mermaid
flowchart TD
    A[Welcome 畫面] --> B[點擊「登入」]
    B --> C[輸入 Email]
    C --> D[輸入密碼]
    D --> E[點擊「登入」]
    E --> F{驗證資料}
    F -->|失敗| G[顯示錯誤訊息]
    G --> C
    F -->|成功| H[呼叫 POST /auth/login]
    H --> I{API 回應}
    I -->|失敗| J[顯示錯誤: 帳密錯誤]
    J --> C
    I -->|成功| K[儲存 JWT Token]
    K --> L[儲存使用者資料]
    L --> M[進入 Dashboard]
```

## 2. 學員新增成績與自動更新PB

### 2.1 新增比賽成績流程

```mermaid
flowchart TD
    A[Dashboard] --> B[點擊 Records Tab]
    B --> C[成績列表頁面]
    C --> D[點擊「+ 新增成績」]
    D --> E[選擇成績類型]
    E --> F{類型?}
    F -->|比賽| G[填寫比賽成績表單]
    F -->|練習| H[填寫練習成績表單]

    G --> I[選擇比賽名稱]
    I --> J[選擇日期]
    J --> K[選擇泳姿]
    K --> L[選擇距離]
    L --> M[選擇池長: 25m/50m]
    M --> N[輸入成績時間]
    N --> O[輸入分鐘]
    O --> P[輸入秒數]
    P --> Q[輸入毫秒]
    Q --> R[新增備註選填]
    R --> S[點擊「儲存」]

    S --> T{驗證資料}
    T -->|失敗| U[顯示錯誤訊息]
    U --> G
    T -->|成功| V[呼叫 POST /results]
    V --> W{API 回應}
    W -->|失敗| X[顯示儲存失敗]
    X --> G
    W -->|成功| Y[顯示「成績已儲存」]
    Y --> Z[系統自動比對 PB]
    Z --> AA{是否打破 PB?}
    AA -->|是| AB[顯示慶祝動畫]
    AB --> AC[顯示「恭喜！新的個人最佳紀錄」]
    AC --> AD[更新 PB 資料]
    AA -->|否| AD
    AD --> AE[返回成績列表]
    AE --> AF[列表顯示新成績]
    AF --> AG[PB 標記顯示]
```

### 2.2 查看 PB 流程

```mermaid
flowchart TD
    A[Dashboard] --> B[點擊「查看全部 PB」]
    B --> C[PB 頁面]
    C --> D[顯示池長切換: 25m/50m]
    D --> E[選擇池長]
    E --> F[呼叫 GET /pb?poolLength=25/50]
    F --> G[顯示各泳姿 PB 卡片]
    G --> H[自由式 50m/100m/200m等]
    G --> I[蝶式 50m/100m等]
    G --> J[仰式 50m/100m等]
    G --> K[蛙式 50m/100m等]
    G --> L[混合式 200m/400m等]

    H --> M[點擊特定距離]
    M --> N[顯示該距離詳情]
    N --> O[顯示最佳時間]
    O --> P[顯示達成日期]
    P --> Q[顯示比賽名稱]
    Q --> R[顯示歷史成績趨勢圖]
    R --> S[可點擊「查看詳細記錄」]
    S --> T[跳轉到該成績詳情頁]
```

## 3. 學員設定與追蹤目標

### 3.1 建立新目標流程

```mermaid
flowchart TD
    A[Dashboard/Targets Tab] --> B[點擊「+ 新增目標」]
    B --> C[選擇泳姿]
    C --> D[選擇距離]
    D --> E[選擇池長]
    E --> F[系統顯示當前 PB]
    F --> G[輸入目標時間]
    G --> H[輸入分鐘]
    H --> I[輸入秒數]
    I --> J[輸入毫秒]
    J --> K{目標時間 < 當前PB?}
    K -->|否| L[顯示警告: 目標應小於當前PB]
    L --> G
    K -->|是| M[選擇到期日期]
    M --> N[輸入備註選填]
    N --> O[點擊「建立目標」]
    O --> P{驗證資料}
    P -->|失敗| Q[顯示錯誤訊息]
    Q --> C
    P -->|成功| R[呼叫 POST /targets]
    R --> S{API 回應}
    S -->|失敗| T[顯示建立失敗]
    T --> C
    S -->|成功| U[顯示「目標已建立」]
    U --> V[返回目標列表]
    V --> W[顯示新目標卡片]
    W --> X[顯示進度條]
    X --> Y[顯示剩餘天數]
```

### 3.2 追蹤目標進度流程

```mermaid
flowchart TD
    A[Targets Tab] --> B[顯示目標列表]
    B --> C[進行中]
    B --> D[已完成]
    B --> E[已逾期]

    C --> F[點擊特定目標]
    F --> G[顯示目標詳情頁]
    G --> H[顯示目標資訊]
    H --> I[泳姿 + 距離 + 池長]
    I --> J[目標時間]
    J --> K[當前 PB]
    K --> L[基準時間 baseline]
    L --> M[計算進度百分比]
    M --> N[progress = baseline-current / baseline-target × 100%]
    N --> O[顯示進度條]
    O --> P[顯示差距時間 current - target]
    P --> Q[顯示到期日與剩餘天數]
    Q --> R[顯示相關成績歷史]

    R --> S[當學員新增成績]
    S --> T{成績項目與目標相符?}
    T -->|否| U[不更新目標]
    T -->|是| V[系統自動更新進度]
    V --> W{達成目標?}
    W -->|是| X[目標狀態改為「已完成」]
    X --> Y[顯示慶祝動畫]
    Y --> Z[發送達成通知]
    W -->|否| AA[更新進度條]
    AA --> AB{檢查到期日}
    AB -->|已過期| AC[狀態改為「已逾期」]
    AB -->|未過期| AD[保持「進行中」]
```

## 4. 學員加入隊伍

### 4.1 透過 QR Code 加入

```mermaid
flowchart TD
    A[Teams Tab/Dashboard] --> B[點擊「加入隊伍」]
    B --> C[顯示加入選項]
    C --> D[選擇「掃描 QR Code」]
    D --> E[開啟相機權限]
    E --> F{權限授予?}
    F -->|否| G[顯示權限說明]
    G --> H[引導至設定頁面]
    F -->|是| I[啟動 QR 掃描器]
    I --> J[掃描教練提供的 QR Code]
    J --> K{掃描成功?}
    K -->|否| L[顯示錯誤: 無效的 QR Code]
    L --> I
    K -->|是| M[解析 Join Code]
    M --> N[顯示隊伍資訊預覽]
    N --> O[隊伍名稱]
    O --> P[教練姓名]
    P --> Q[隊伍簡介]
    Q --> R[點擊「確認加入」]
    R --> S[呼叫 POST /teams/join]
    S --> T{API 回應}
    T -->|失敗| U[顯示錯誤訊息]
    U --> B
    T -->|成功| V[顯示「申請已送出」]
    V --> W[狀態: 待審核]
    W --> X[等待教練核准]
    X --> Y[收到核准通知]
    Y --> Z[隊伍顯示在「我的隊伍」]
```

### 4.2 透過邀請碼加入

```mermaid
flowchart TD
    A[Teams Tab] --> B[點擊「加入隊伍」]
    B --> C[選擇「輸入邀請碼」]
    C --> D[顯示輸入框]
    D --> E[輸入 6-8 位邀請碼]
    E --> F[點擊「查詢」]
    F --> G{驗證格式}
    G -->|失敗| H[顯示錯誤: 格式不正確]
    H --> E
    G -->|成功| I[呼叫 GET /teams/by-code/:code]
    I --> J{API 回應}
    J -->|失敗| K[顯示錯誤: 無效或過期]
    K --> E
    J -->|成功| L[顯示隊伍資訊]
    L --> M[確認加入流程]
    M --> N[同 QR Code 後續流程]
```

## 5. 教練建立隊伍與產生邀請

### 5.1 建立隊伍流程

```mermaid
flowchart TD
    A[Teams Tab - 教練視角] --> B[點擊「+ 建立隊伍」]
    B --> C[填寫隊伍資訊]
    C --> D[輸入隊伍名稱必填]
    D --> E[輸入隊伍簡介選填]
    E --> F[選擇隊伍類型]
    F --> G[俱樂部/學校/私人教學]
    G --> H[上傳隊伍照片選填]
    H --> I[設定是否公開]
    I --> J[點擊「建立」]
    J --> K{驗證資料}
    K -->|失敗| L[顯示錯誤訊息]
    L --> C
    K -->|成功| M[呼叫 POST /teams]
    M --> N{API 回應}
    N -->|失敗| O[顯示建立失敗]
    O --> C
    N -->|成功| P[顯示「隊伍已建立」]
    P --> Q[進入隊伍管理頁]
    Q --> R[顯示空的成員列表]
    R --> S[提示「開始邀請成員」]
```

### 5.2 產生邀請碼與 QR Code

```mermaid
flowchart TD
    A[隊伍管理頁] --> B[點擊「邀請成員」]
    B --> C[顯示邀請選項]
    C --> D[選擇邀請方式]
    D --> E[產生新邀請碼]
    D --> F[使用現有邀請碼]

    E --> G[呼叫 POST /teams/:id/join-codes]
    G --> H[設定邀請碼參數]
    H --> I[選擇有效期限]
    I --> J[7天/30天/永久]
    J --> K[設定使用次數限制]
    K --> L[無限制/10次/50次/100次]
    L --> M[點擊「產生」]
    M --> N{API 回應}
    N -->|失敗| O[顯示錯誤]
    N -->|成功| P[顯示邀請碼]
    P --> Q[8位英數字碼]
    Q --> R[自動產生 QR Code]
    R --> S[顯示分享選項]
    S --> T[複製邀請碼]
    S --> U[分享 QR Code 圖片]
    S --> V[分享連結]

    U --> W[儲存到相簿]
    U --> X[分享到通訊軟體]
    V --> Y[產生 Deep Link]
    Y --> Z[aquatrack://join/:code]
```

## 6. 教練審核學員

### 6.1 審核申請流程

```mermaid
flowchart TD
    A[教練 - Teams Tab] --> B[顯示隊伍列表]
    B --> C[點擊特定隊伍]
    C --> D[隊伍管理頁]
    D --> E{有待審核申請?}
    E -->|是| F[顯示紅點通知]
    E -->|否| G[無待審核]

    F --> H[點擊「待審核」標籤]
    H --> I[顯示申請列表]
    I --> J[學員頭像]
    J --> K[學員姓名]
    K --> L[申請時間]
    L --> M[點擊特定申請]
    M --> N[顯示學員詳細資料]
    N --> O[個人簡介]
    O --> P[生日 / 年齡]
    P --> Q[性別]
    Q --> R[主要泳姿]
    R --> S[歷史成績預覽可選]

    S --> T[審核選項]
    T --> U[點擊「核准」]
    T --> V[點擊「拒絕」]

    U --> W[確認對話框]
    W --> X[點擊「確認核准」]
    X --> Y[呼叫 POST /teams/:id/approve]
    Y --> Z{API 回應}
    Z -->|失敗| AA[顯示錯誤]
    Z -->|成功| AB[顯示「已核准」]
    AB --> AC[學員移至「成員」列表]
    AC --> AD[學員收到核准通知]

    V --> AE[輸入拒絕原因選填]
    AE --> AF[呼叫 POST /teams/:id/reject]
    AF --> AG{API 回應}
    AG -->|失敗| AH[顯示錯誤]
    AG -->|成功| AI[顯示「已拒絕」]
    AI --> AJ[申請從列表移除]
    AJ --> AK[學員收到拒絕通知]
```

### 6.2 管理現有成員

## 7. 教練建立與指派課表

```mermaid
flowchart TD

    G --> H[輸入課表名稱]
    H --> I[輸入課表說明]
    I --> J[選擇難度等級]
    J --> K[初級/中級/高級]
    K --> L[點擊「下一步」]

    L --> M[新增訓練組 Set]
    M --> N[點擊「+ 新增組別」]
    N --> O[填寫組別資訊]
    O --> P[選擇泳姿]
    P --> Q[輸入距離 米]
    Q --> R[輸入趟數]
    R --> S[輸入目標時間選填]
    S --> T[輸入休息時間 秒]
    T --> U[輸入備註選填]
    U --> V[儲存組別]

    V --> W{繼續新增組別?}
    W -->|是| N
    W -->|否| X[調整組別順序]
    X --> Y[拖曳排序]
    Y --> Z[點擊「完成」]

    AB -->|成功| AD[顯示「課表已建立」]

```

### 7.2 指派課表流程

```mermaid
    N --> O[點擊「確認指派」]

    O --> P[呼叫 POST /plans/:id/assignments]
    P --> Q{API 回應}
    Q -->|失敗| R[顯示錯誤]
    Q -->|成功| S[顯示「已指派」]
    S --> T[被指派學員收到通知]
    T --> U[課表出現在學員的訓練列表]
```

```mermaid
    B --> C[今日訓練]

    C --> F[點擊特定訓練]
```

### 8.2 回報訓練完成流程

```mermaid
flowchart TD
    A[訓練執行模式] --> B[顯示當前組別]
    B --> C[倒數計時器選用]
    C --> D[完成當前組別]
    D --> E[點擊「完成」]
    E --> F[進入下一組]
    F --> G{所有組別完成?}
    G -->|否| B
    G -->|是| H[顯示回饋表單]

    H --> I[輸入完成度 %]
    I --> J[拖曳滑桿 0-100%]
    J --> K[輸入 RPE 體感強度]
    K --> L[1-10 分]
    L --> M[輸入訓練備註]
    M --> N[感覺如何/困難點]
    N --> O[點擊「提交回饋」]

    O --> P[呼叫 POST /plans/:id/feedback]
    P --> Q{API 回應}
    Q -->|失敗| R[顯示錯誤]
    Q -->|成功| S[顯示「感謝回饋」]
```

## 附錄：關鍵使用者體驗設計原則

### 1. 即時回饋
- 任何資料提交後 200ms 內顯示載入狀態
- 成功/失敗訊息清晰明確
- 打破 PB 時有視覺慶祝效果

### 2. 減少輸入負擔
- 使用選擇器而非手動輸入（泳姿、距離）
- 記住上次輸入的值
- 提供快速填入選項

### 3. 防呆設計
- 目標時間必須小於當前 PB
- 刪除/移除需二次確認
- 重要操作前預覽結果

### 4. 離線優先
- 關鍵資料本地快取
- 離線時可查看歷史記錄
- 網路恢復後自動同步

### 5. 通知策略
- 核准/拒絕即時推送
- 打破 PB 發送通知
- 目標達成/逾期提醒
- 教練指派新訓練通知
